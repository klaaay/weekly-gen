### 概述总结
Server-Driven UI 是一种通过 API 控制客户端布局的模式，核心是使用 JSON 树描述界面，客户端通过组件注册表和渲染器将其映射为原生组件，从而减少跨平台开发成本并实现快速迭代。

- 📝 **核心概念**：Server-Driven UI 将布局决策从客户端移至 API，通过 JSON 树定义界面结构，客户端仅负责渲染已注册的组件。
- 🧩 **三部分架构**：模式包含 JSON 树（定义布局和内容）、组件注册表（映射组件名称到实际组件）和渲染器（递归渲染节点）。
- 🏗️ **JSON 树结构**：每个节点包含 `type`（组件名）、`props`（配置数据）、`children`（子节点）和 `actions`（交互行为），形成完整契约。
- 🗺️ **组件注册表**：客户端维护一个从字符串到组件的映射，确保服务器只能请求已存在的组件，避免运行时引入新功能。
- 🔄 **渲染器实现**：递归遍历 JSON 树，查找组件并传递 props 和 children，支持错误边界和懒加载以增强健壮性。
- 🎯 **行为数据化**：Actions 将交互行为（如导航、跟踪事件）表示为数据，通过统一调度器处理，使组件无需关心具体行为逻辑。
- ✅ **适用场景**：适合频繁变化的页面（如推荐、促销），可快速进行布局实验；不适合认证、离线场景。
- ⚖️ **平台合规性**：Apple 和 Google 允许 SDUI，只要不改变应用主要功能、不绕过安全机制或创建应用商店；组件注册确保合规。
- ⚠️ **注意事项**：版本控制（避免破坏旧客户端）、缓存策略（避免个性化缓存爆炸）、以及组件变更时的向后兼容性。
