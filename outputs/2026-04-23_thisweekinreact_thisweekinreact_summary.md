### [AI 生成界面默认不可访问——前端大师博客](https://frontendmasters.com/blog/ai-generated-ui-is-inaccessible-by-default/)

**原文标题**: [AI-Generated UI Is Inaccessible by Default – Frontend Masters Blog](https://frontendmasters.com/blog/ai-generated-ui-is-inaccessible-by-default/)

AI 生成的 React 组件在默认情况下往往缺乏可访问性，因为它们主要优化视觉输出而忽略了语义信息，导致屏幕阅读器等辅助技术无法正确识别。本文提出了一个五层强制系统来确保语义正确性。

- 🧠 **问题根源**：AI 训练数据中语义 HTML 和 ARIA 属性不足，导致生成的代码多为视觉导向的`<div>`元素，缺乏可访问性所需的角色、状态和键盘支持。
- 🌳 **可访问性树**：浏览器构建的可访问性树与渲染树分离，依赖 HTML 语义和 ARIA 属性来为辅助技术提供信息，而 AI 生成的代码往往无法正确构建此树。
- 📝 **提示词约束**：通过在工作区配置中设置严格的生成规则，强制 AI 使用语义正确的 HTML 元素和 ARIA 属性，例如用`<button>`替代`<div onClick>`。
- 🔍 **静态分析**：使用 ESLint 插件`eslint-plugin-jsx-a11y`在代码提交前检测并阻止可访问性违规，如缺少键盘事件或错误角色。
- 🧪 **运行时测试**：结合 Jest 和 Playwright 进行浏览器测试，使用`axe-core`自动检测可访问性漏洞，并验证键盘导航和动态状态。
- 🔗 **CI 集成**：在持续集成流程中自动运行可访问性检查，确保代码合并前通过所有测试，防止有问题的组件进入生产环境。
- 🧩 **组件抽象**：采用如 Headless UI、Radix UI 等库，这些库内置可访问性合约，减少 AI 生成原始交互元素的需求，将语义层与视觉层解耦。
- ⚠️ **常见失败模式**：AI 常生成无标签图标按钮、无焦点陷阱的模态框、自定义选择器等复杂组件，这些都需要额外的手动修复或使用专用库。
- ⏱️ **成本与限制**：添加可访问性约束初期会增加 3-8 分钟/组件，但远比事后修复（45-90 分钟/组件）高效；自动测试可覆盖 70-85% 的问题，其余需手动测试。

### [Margelo 如何帮助 Discord 提升 React Native 新架构性能](https://blog.margelo.com/margelo-discord-react-native-performance)

**原文标题**: [How Margelo Helped Discord Improve React Native's New Architecture Performance](https://blog.margelo.com/margelo-discord-react-native-performance)

Discord 在将 Android 应用迁移到 React Native 新架构时，动画性能出现严重卡顿。经过深入分析，发现问题的核心在于 Reanimated 库与新架构的交互方式，特别是其更新机制导致大量不必要的 Shadow Tree 节点克隆和布局计算，从而拖慢了动画表现。通过优化更新同步机制和重新启用非布局属性的快速更新路径，最终成功将卡顿帧率降低了 26%，恢复了流畅的动画体验。

- 🎯 **问题定位**：Discord 迁移至 React Native 新架构后，动画出现严重卡顿，类似“2006 年 PowerPoint 幻灯片”效果。
- 🔍 **根本原因**：Reanimated 在处理动画更新时，会克隆大量 Shadow Tree 节点，即使只有少数视图在动画，导致性能瓶颈。
- ⚙️ **更新机制**：动画通过共享值驱动，更新从 JavaScript 传递到 C++ 运行时，最终通过 `shadowTree.commit()` 应用到 Shadow Tree。
- 🌳 **Shadow Tree 作用**：新架构中，Shadow Tree 是原生视图与 React 状态同步的唯一来源，任何更新都必须通过它进行。
- 🚧 **性能瓶颈**：Reanimated 的 `cloneShadowTreeWithNewProps` 函数在每次动画或 React 更新时克隆大量节点，且即使是非布局属性（如透明度）也会触发昂贵的布局计算。
- 🔄 **同步问题**：Reanimated 直接修改 Shadow Tree 可能导致与 React 状态不同步，需通过提交钩子（commit hook）强制同步，进一步增加开销。
- 💡 **解决方案**：引入 `FORCE_REACT_RENDER_FOR_SETTLED_ANIMATIONS` 标志，动画结束后同步回 React，及时清理未动画节点；重新启用 `ANDROID_SYNCHRONOUSLY_UPDATE_UI_PROPS` 快速路径，优化非布局属性更新。
- 📈 **实际效果**：优化后，Discord Android 应用的卡顿帧率降低 26%，动画恢复流畅。
- 🔮 **未来展望**：Reanimated 将逐步采用 React Native 核心的“共享动画后端”，进一步提升动画性能。

- 🛠️ **工具链优化**：升级 zlib 依赖至 zlib-ng，gzip 压缩速度提升最高达 5.5 倍；`bunx claude` 现在可作为 `bunx @anthropic-ai/claude-code` 的快捷方式。
- 🐛 **大量错误修复**：修复了涉及 Node.js 兼容性、Bun API、Web API、`bun install`、JavaScript 打包器、CSS 解析器、`bun test`、Windows 平台、JavaScript 引擎以及内部运行时的数百个问题，提升了稳定性和兼容性。
