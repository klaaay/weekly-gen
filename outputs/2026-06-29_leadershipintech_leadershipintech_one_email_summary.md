### [包管理器常见弱点 | Andrew Nesbitt](https://nesbitt.io/2026/05/04/package-manager-cwes.html)

**原文标题**: [Package Manager CWEs | Andrew Nesbitt](https://nesbitt.io/2026/05/04/package-manager-cwes.html)

本报告系统梳理了包管理器客户端与注册服务端历史上出现的安全漏洞模式，总结了约 20 种反复出现的失败模式，并提供了针对性的防御建议。

- 📦 **客户端：归档解压路径穿越** - 攻击者利用 `../`、符号链接或 Windows 反斜杠绕过路径检查，将文件写入目标目录之外。建议：解压后校验每个文件的解析路径是否在目标目录内，并处理所有操作系统分隔符。
- 🔧 **客户端：VCS 参数注入** - 用户控制的字符串（如 Git URL 或分支名）被当作命令行选项传递给 `git clone`，导致远程代码执行。建议：使用 `--` 分隔符并白名单允许的选项。
- 🔐 **客户端：完整性校验失效** - 签名文件缺失、校验错误被当作成功、或哈希检查在缓存路径中被跳过。建议：测试签名不存在、格式错误、内容不匹配三种情况，均拒绝安装。
- 🚫 **客户端：凭证泄露给错误目标** - 认证令牌通过重定向、镜像或依赖项被发送到非预期的服务器，或凭证被写入日志/发布包。建议：凭证绑定到来源，跨域重定向时剥离，并测试 `.env` 不被打包。
- 🎯 **客户端：依赖源选择错误** - 私有包名与公共注册表冲突，解析器选择版本更高的公共包（依赖混淆）。建议：每个依赖从唯一源解析，并在锁文件中固定。
- 📂 **客户端：本地文件/目录被当作配置** - 在不受信任的目录中运行包管理器时，当前目录下的可执行文件或配置文件被自动加载。建议：避免从工作目录隐式加载配置。
- 🗂️ **客户端：共享文件系统权限问题** - 可预测的临时文件路径、全局可写的缓存目录、或未遵循 umask 的文件权限，导致本地用户劫持。建议：使用 `mkdtemp` 创建进程级临时目录，创建时即设置权限。
- ⚠️ **客户端：反序列化漏洞** - 使用不安全的 YAML 或 XML 解析器处理包元数据，导致在安装前即可执行代码或泄露文件。建议：使用安全加载变体，关闭 DTD 和外部实体。
- 💥 **客户端：资源耗尽攻击** - 压缩炸弹、正则表达式回溯、深度 JSON Schema 引用导致磁盘或内存耗尽。建议：限制解压大小、递归深度，并审查处理元数据的正则表达式。
- 🖥️ **客户端：终端转义序列注入** - 包名、描述中的 ANSI 转义码被直接输出到终端，可篡改输出或执行恶意命令。建议：输出到 stdout 前剥离控制字符。
- 🔒 **客户端：锁文件/缓存未真正锁定** - 锁文件存在但某些代码路径忽略它，或解析器差异导致安装的包与审计工具看到的包不同。建议：确保所有安装路径都遵守锁文件。
- 🛡️ **客户端：构建沙箱逃逸** - 试图隔离构建环境的工具存在漏洞，如文件描述符泄露、权限未正确丢弃。建议：若实现沙箱，需全面审计隔离机制。
- 🧩 **客户端：归档解析器内存损坏** - C 语言编写的归档解析器存在整数溢出、越界写入等漏洞。建议：使用内存安全语言或经过充分模糊测试的库。
- 🏢 **注册服务端：发布/替换他人包** - 授权检查缺陷导致攻击者可覆盖或删除他人包。建议：对发布、撤回、所有权转移端点进行最严格的威胁建模。
- 🔑 **注册服务端：账户接管** - 维护者邮箱域名过期、密码复用、弱随机令牌等导致账户被劫持。建议：检测域名过期风险，强制发布时启用 2FA，并限定 API 令牌范围。
- 🌐 **注册服务端：存储型 XSS** - 渲染包 README、描述时未正确转义，导致攻击者在登录用户浏览器中执行脚本。建议：使用真正的消毒器并配合 CSP。
- 💻 **注册服务端：服务器端代码执行** - 注册表解析包元数据或克隆仓库时，复用客户端的漏洞（如 YAML 反序列化、参数注入）。建议：检查所有客户端解析漏洞在服务端的对应情况。
- 🌍 **注册服务端：SSRF** - 用户提供的 URL 被注册表服务端获取，可扫描内网或访问元数据服务。建议：白名单允许的域名，检查 DNS 重绑定，阻止私有地址范围。
- 🆔 **注册服务端：IDOR** - 操作对象 ID 时仅检查操作权限，未验证所有权，导致越权访问其他组织的包或配置。建议：每个处理对象 ID 的处理器都需检查所有权。
- 🎭 **注册服务端：令牌权限夸大** - “只上传”令牌实际具有会话权限，或只读令牌可交换为全范围令牌。建议：令牌声明的范围必须与服务端强制执行的范围一致，且无窄令牌扩权路径。
- 🔄 **注册服务端：客户端与服务端元数据不一致** - 注册表展示的元数据与客户端安装时使用的不同，导致审计工具与安装结果不一致。建议：确保包内容的单一事实来源，所有消费者都从该来源读取。
