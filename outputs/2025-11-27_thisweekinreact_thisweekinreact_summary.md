### [我们如何意外地通过追求正确性提升了路由匹配性能 | TanStack 博客](https://tanstack.com/blog/tanstack-router-route-matching-tree-rewrite)

**原文标题**: [How we accidentally made route matching more performant by aiming for correctness | TanStack Blog](https://tanstack.com/blog/tanstack-router-route-matching-tree-rewrite)

TanStack Router 通过重构路由匹配算法，在追求正确性的同时意外实现了性能的指数级提升，将匹配复杂度从依赖路由数量转变为依赖路径段数量，并采用段前缀树、位掩码和缓存等优化技术。

- 🚀 性能提升：路由匹配性能提升最高达20000倍，小型应用提升60倍，大型应用提升10000倍
- 🎯 问题根源：原算法基于扁平化路由列表排序匹配，存在弱排序不一致和跨浏览器兼容性问题
- 🌳 数据结构：采用段前缀树替代扁平列表，将复杂度从O(N)降为O(M)，N为路由数量，M为路径段数
- 🔄 遍历优化：使用反向栈处理实现深度优先搜索，优先处理高优先级路由段
- 💾 内存优化：通过位掩码技术记录可选段状态，避免数组拷贝带来的内存分配
- ⚡ 解析优化：重用类型化数组进行路径段解析，减少对象创建开销
- 🗂️ 缓存机制：采用LRU缓存匹配结果，防止内存无限增长的同时保持O(1)读取性能
- 📊 实际影响：完整路由操作（包含验证、中间件等）性能提升更为温和但依然显著

### [全新CSS子网格布局设计 • 乔希·W·科莫](https://www.joshwcomeau.com/css/subgrid/)

**原文标题**: [Brand New Layouts with CSS Subgrid • Josh W. Comeau](https://www.joshwcomeau.com/css/subgrid/)

CSS Subgrid 是一种扩展 CSS Grid 布局的新功能，允许子元素继承父网格的布局结构，从而创建更灵活和一致的 UI 设计。它解决了嵌套元素无法直接参与网格布局的问题，并支持动态响应内容变化，同时提供了调试工具和浏览器兼容性解决方案。

- 🎯 **子网格基础**：通过 `grid-template-rows: subgrid` 和 `grid-template-columns: subgrid` 继承父网格的布局，使嵌套元素（如 `<ul>` 中的 `<li>`）能够对齐到同一网格线。
- 🚀 **新布局可能**：子网格允许兄弟元素相互响应，例如在卡片布局中动态调整图像和文本列宽，避免内容溢出并优化空间利用。
- ⚠️ **行空间预留**：子网格默认仅占用单行，需使用 `grid-row: span N` 显式指定多行范围，否则内容会压缩到同一行。
- 🔢 **行号重置**：子网格中的网格线编号会重新从 1 开始，但命名网格区域仍可从父网格访问。
- 📏 **流体布局限制**：子网格与 `auto-fill`/`auto-fit` 等流体网格设计不兼容，需明确指定列数。
- 🌐 **浏览器兼容**：子网格自 2023 年起获主流浏览器支持，但使用 `@supports` 查询为旧浏览器提供降级布局（如垂直堆叠）。
- 🛠️ **开发工具**：Firefox、Chrome 和 Safari 的开发者工具支持可视化子网格结构，便于调试。
- 💡 **实用案例**：如 Stripe 开发者网站所示，子网格可用于构建复杂整体布局，或逐步应用于特定组件以提升一致性。
