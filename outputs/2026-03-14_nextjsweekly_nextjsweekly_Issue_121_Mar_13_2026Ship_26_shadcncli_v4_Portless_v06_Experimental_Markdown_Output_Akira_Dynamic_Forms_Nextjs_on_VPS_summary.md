### [在 Vercel 上无限扩展重定向 - Vercel](https://vercel.com/blog/scaling-redirects-to-infinity-on-vercel)

**原文标题**: [Scaling redirects to infinity on Vercel - Vercel](https://vercel.com/blog/scaling-redirects-to-infinity-on-vercel)

Vercel 为应对大规模重定向带来的延迟和成本问题，开发了新的批量重定向系统，通过分片、布隆过滤器和二进制搜索等技术，实现了高效、低延迟的重定向处理。

- 🚀 **优化目标**：支持数百万静态重定向，确保无重定向项目零延迟成本，快速处理“无重定向”请求，并降低进程内存使用。
- 🛠️ **初始设计**：结合重定向数据和布隆过滤器，使用 JSONL 文件格式存储，通过布隆过滤器快速排除无重定向请求，减少不必要的 JSON 解析。
- 📊 **分片策略**：通过哈希将重定向路径分布到多个小分片中，避免加载整个数据集，结合布隆过滤器进一步降低内存和查找开销。
- 🔍 **查找流程**：请求时先检查布隆过滤器，仅当可能匹配时才加载对应分片，避免解析整个 JSON 文件，提升“无重定向”路径效率。
- ⚡ **性能瓶颈**：JSON 解析在正匹配时成为延迟瓶颈，尤其在 CPU 高负载下导致延迟峰值，需优化数据格式。
- 🔄 **二进制搜索优化**：将重定向按键排序存储，实现对数时间搜索，仅解析特定重定向的 JSON，消除解析整个分片的开销，稳定延迟。
- 📉 **效果显著**：优化后重定向查找延迟大幅下降，CPU 高负载下的延迟峰值消失，系统性能更可预测。
- 🧩 **设计原则**：针对大规模冷数据和严格延迟要求，采用分片、布隆过滤器和二进制搜索的组合方案，避免过度优化，通过迭代改进。

### [理解 React Fiber 存在的意义 | 深入 React 内部](https://inside-react.vercel.app/blog/understanding-why-react-fiber-exists)

**原文标题**: [Understanding Why React Fiber Exists | Inside React](https://inside-react.vercel.app/blog/understanding-why-react-fiber-exists)

React Fiber 是为了解决 React 15 中递归渲染导致的性能问题而引入的架构，它通过将渲染工作拆分为可中断和可恢复的小单元，实现了时间切片和优先级调度，从而提升用户交互的响应性。

- 🧵 **摆脱 JavaScript 调用栈的限制**：React 15 使用递归渲染，一旦开始就无法中断，导致长时间任务阻塞用户交互。
- ⏸️ **实现可中断的渲染过程**：Fiber 将渲染拆分为小工作单元，允许 React 在每约 5 毫秒后暂停，将控制权交还浏览器处理高优先级事件。
- 🔗 **Fiber 作为链表数据结构**：每个 Fiber 节点代表 UI 的一部分，通过 child、sibling 和 return 指针连接，形成可手动遍历和暂停的数据结构。
- 🎯 **支持优先级调度**：Fiber 允许 React 区分更新优先级（如用户输入优于后台数据获取），确保关键交互得到即时响应。
- ⏱️ **时间切片提升用户体验**：通过分段渲染，React 在后台逐步处理大型更新（如渲染 500 个产品），同时保持用户输入和动画的流畅性。
- 🔄 **保持 React 核心哲学不变**：Fiber 在底层优化调度，但组件仍保持为状态的纯函数，维护了开发者的简单心智模型和向后兼容性。
