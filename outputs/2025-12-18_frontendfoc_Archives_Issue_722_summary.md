### [h1 元素的默认样式正在改变](https://developer.mozilla.org/en-US/blog/h1-element-styles/)

**原文标题**: [Default styles for h1 elements are changing](https://developer.mozilla.org/en-US/blog/h1-element-styles/)

浏览器正逐步取消嵌套标题的默认用户代理样式调整，开发者需检查网站是否依赖这些样式以避免显示异常和 Lighthouse 检测失败。本文介绍了相关变更内容、影响识别方法及优化建议。

- 🔄 浏览器将移除 `<h1>` 在 `<section>`、`<article>`、`<nav>`、`<aside>` 中的默认样式继承规则，不再自动匹配 `<h2>`/`<h3>` 等样式

- 🛠️ 修复建议：为 `<h1>` 显式设置 `font-size: 2em; margin-block: 0.67em;`，推荐使用 `:where()` 降低选择器优先级
- 📝 规范实践：始终使用 `<h2>`-`<h6>` 构建标题层级，避免依赖浏览器默认样式，可参考 MDN 最新使用说明更新 CSS 重置方案

### [全新布局：CSS 子网格 • Josh W. Comeau](https://www.joshwcomeau.com/css/subgrid/)

**原文标题**: [Brand New Layouts with CSS Subgrid • Josh W. Comeau](https://www.joshwcomeau.com/css/subgrid/)

CSS Subgrid 是 CSS Grid 布局的新特性，它允许网格布局通过 DOM 树向下延伸，使嵌套元素也能参与父网格的布局，从而开启全新的 UI 设计可能性。本文通过示例介绍了 Subgrid 的基本用法、优势、常见陷阱及浏览器兼容性。

- 🚀 **扩展网格布局**：Subgrid 允许嵌套元素（如 `<ul>` 中的 `<li>`）继承并参与父网格的布局结构，解决了以往只有直接子元素才能使用网格的限制。
- 🎨 **实现布局对齐**：通过 Subgrid，可以使多个嵌套网格共享同一网格模板，确保内容（如图片与文本）在不同卡片间对齐，提升视觉一致性。
- ⚠️ **注意行数预留**：使用 Subgrid 时，必须显式指定子网格跨越的行数（如 `grid-row: span 6`），否则内容可能被压缩到单一行中。
- 🔢 **网格线编号重置**：子网格会重新索引继承的网格线，父网格的第 2 行在子网格中变为第 1 行，但命名网格区域仍可访问。
- 🌊 **不兼容流体网格**：Subgrid 目前无法与 `auto-fill`/`auto-fit` 等流体布局模式结合使用，需明确指定列数。
- 🛡️ **提供浏览器回退**：使用 `@supports` 特性查询为不支持 Subgrid 的浏览器提供备用布局（如单列堆叠），确保基本体验。
- 🔧 **强大开发工具**：现代浏览器（如 Firefox、Chrome、Safari）提供了网格开发工具，可可视化调试 Subgrid 布局结构。
- 💡 **逐步采用优势**：Subgrid 无需重构整个项目，可渐进式用于解决特定布局问题，如复杂卡片对齐或动态内容响应。

### [物品流，第一部分：布局的新统一概念 | WebKit](https://webkit.org/blog/16587/item-flow-part-1-a-new-unified-concept-for-layout/)

**原文标题**: [  Item Flow, Part 1: A new unified concept for layout | WebKit](https://webkit.org/blog/16587/item-flow-part-1-a-new-unified-concept-for-layout/)

CSS 布局领域正在探索一种名为“Item Flow”的新统一概念，旨在整合 Flexbox 和 Grid 的部分功能，并可能为瀑布流（masonry）布局提供原生支持。这一提案源于对现有布局工具局限性的讨论，试图通过一组新的属性来简化并增强布局控制。

- 🧩 **整合布局系统**：Item Flow 旨在统一 Flexbox 的 `flex-flow` 和 Grid 的 `grid-auto-flow`，提出新的属性集（如 `item-direction`、`item-wrap`、`item-pack`、`item-slack`），以简化布局控制并扩展功能。
- 🔄 **方向与换行控制**：`item-direction` 定义内容流动方向（行或列），`item-wrap` 控制是否换行，并为 Grid 新增 `nowrap` 选项，使其能实现单行布局。
- 📦 **包装模式扩展**：`item-pack` 引入新的包装选项，如 `dense`（密集包装，可能用于 Flexbox）、`balance`（平衡行间项目分布），以及 `collapse`（可能作为瀑布流布局的触发器）。
- ⚖️ **容差与灵活性**：`item-slack`（或类似属性）允许设置布局容差，控制项目放置的严格程度，例如在接近空间阈值时决定是否换行或压缩项目。
- 🚀 **潜在新功能**：提案可能为 Flexbox 和 Grid 带来新能力，如 Grid 的禁止换行、Flexbox 的密集包装，以及更智能的项目平衡布局，解决现有痛点。
- 💬 **开放讨论阶段**：Item Flow 仍处于草案阶段，名称和具体行为可能调整，开发者可通过 CSS 工作组或社交媒体反馈意见，影响最终标准制定。
