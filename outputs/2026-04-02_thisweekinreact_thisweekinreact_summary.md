Signals 是响应式编程的核心机制，通过 push-pull 算法实现细粒度数据更新。push 机制在信号变更时主动通知依赖项，pull 机制在计算值被读取时才惰性重新计算。系统通过全局 STACK 自动追踪依赖关系，结合脏标记缓存实现高效更新。

- 🚀 **Push 机制**：信号变更时立即向订阅者推送通知，实现主动传播
- 🛋️ **Pull 机制**：计算值仅在读取时重新计算，支持惰性求值与缓存
- 🧲 **自动依赖追踪**：通过全局 STACK 在运行时动态建立信号与计算值间的关联
- 🧠 **脏标记系统**：计算值被标记为脏时才会重新计算，避免不必要的更新
- 🌳 **依赖图管理**：自动清理无效依赖，保持响应式图的实时准确性
- 🔄 **双向传播**：push 向下传播失效通知，pull 向上触发重新计算
- 🏗️ **框架应用**：已被 Solid、Vue、Preact 等现代前端框架广泛采用
- 📜 **标准化进程**：TC39 提案正在推动 Signals 成为 JavaScript 原生特性

### [GitHub - chenglou/pretext · GitHub](https://github.com/chenglou/pretext)

**原文标题**: [GitHub - chenglou/pretext · GitHub](https://github.com/chenglou/pretext)

Pretext 是一个纯 JavaScript/TypeScript 库，用于多行文本的测量与布局，无需依赖 DOM 测量，避免触发浏览器重排，支持多种语言和渲染目标。

- 📏 **高效文本测量**：通过 `prepare()` 进行一次性文本分析与测量，再通过 `layout()` 进行快速高度计算，避免重复 DOM 操作。
- 🚀 **性能优异**：在基准测试中，`prepare()` 约 19ms 处理 500 条文本，`layout()` 仅约 0.09ms。
- 🌍 **多语言支持**：支持包括表情符号和混合双向文本在内的多种语言，并适配浏览器特定行为。
- 🛠️ **灵活布局 API**：提供 `layoutWithLines()`、`walkLineRanges()` 和 `layoutNextLine()` 等方法，支持手动控制行布局和动态宽度调整。
- 🎯 **应用场景广泛**：适用于虚拟化布局、用户自定义布局（如瀑布流）、开发时验证文本溢出及防止布局偏移等。
- ⚠️ **注意事项**：目前主要针对常见文本设置（如 `white-space: normal`），在 macOS 上使用 `system-ui` 字体可能影响精度，需使用具体字体名称。

- 🚀 **避免布局重排**：通过 canvas.measureText() 测量文本并缓存宽度，将布局计算转为纯算术操作，消除强制布局重排导致的性能卡顿。

- 🖼️ **Canvas 渲染的权衡**：虽然支持 Canvas/SVG/WebGL 渲染实现视觉效果，但会丧失无障碍访问能力，适用于游戏或设计工具等特定场景。
- 🔍 **社区关注偏差**：炫酷的演示易传播，但库的核心价值在于不可见的性能优化，应优先关注其测量引擎而非视觉效果。
