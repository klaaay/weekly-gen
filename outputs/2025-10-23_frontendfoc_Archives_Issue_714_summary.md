### [双按钮问题 – Frontend Masters 博客](https://frontendmasters.com/blog/the-two-button-problem/)

**原文标题**: [The Two Button Problem – Frontend Masters Blog](https://frontendmasters.com/blog/the-two-button-problem/)

文章讨论了界面设计中常见的"双按钮问题"，即两个按钮同时存在时用户难以辨别当前激活状态的设计缺陷。

- 🎯 双按钮设计常导致用户无法区分哪个按钮处于激活状态，尤其在遥控器和键盘操作场景中更为明显

- 🎨 解决方案包括统一按钮基础样式，仅对激活状态添加明显视觉差异
- 📱 可通过CSS媒体查询针对不同设备类型应用更明确的状态指示设计
- 🔄 切换按钮设计中常见将激活按钮灰显的反模式，反而降低了可识别性
- 💡 建议采用拟物化设计思路，通过阴影变化表现按钮的按下状态
- ⚠️ 需区分"已激活"和"即将激活"两种不同状态的设计表达
- 🎪 作者强调双按钮场景不能随意使用两种不同样式，必须确保状态明确可辨

### [@starting-style - CSS | MDN](https://developer.mozilla.org/en-US/docs/Web/CSS/@starting-style)

**原文标题**: [@starting-style - CSS | MDN](https://developer.mozilla.org/en-US/docs/Web/CSS/@starting-style)

CSS @starting-style 规则用于定义元素首次显示时的初始样式，以便实现平滑的过渡动画效果。

- 🎯 **用途**：为元素首次渲染或从 `display: none` 变为可见时提供初始样式，支持 CSS 过渡动画
- 📝 **语法**：支持独立代码块或嵌套在现有规则集中，定义起始属性值
- ⚡ **触发条件**：元素首次加入 DOM 或从隐藏状态变为可见时应用起始样式
- 🎪 **应用场景**：特别适用于顶层元素（如弹窗、对话框）、显示/隐藏切换和 DOM 添加/删除动画
- 🔄 **状态管理**：涉及起始样式、过渡状态和默认状态三种样式状态
- 💡 **注意事项**：需在原始规则之后定义，确保起始样式生效；仅适用于 CSS 过渡，动画无需此规则
- 🌟 **兼容性**：自 2024 年 8 月起支持最新设备和浏览器版本

- 🔒 **来源认证与隐私保护**：内容经加密签名后分发，浏览器可验证来源真实性，用户隐私信息在跳转前不会泄露
- 📦 **标准化封装格式**：采用二进制文件封装HTTP请求响应及数字签名，支持最长7天有效期
- 🛠️ **多场景应用支持**：适用于离线浏览、第三方缓存服务，并与AMP、Web Bundles等技术兼容
- ⚡ **企业级性能提升**：实际案例显示LCP提升最高达41%，会话量增长27%，页面加载时间减少21%
- 🔧 **多样化工具生态**：提供Cloudflare Workers、NGINX模块、Envoy过滤器等平台专用工具链
- 🌍 **搜索引擎集成**：Google搜索优先抓取SXG内容，通过预加载机制实现近瞬时页面加载
