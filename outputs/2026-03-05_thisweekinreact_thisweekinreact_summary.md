Cloudflare 的 AI 生成 Next.js 替代框架 vinext 在快速开发过程中，因缺乏安全测试而存在多个严重漏洞，包括会话劫持、缓存污染和中间件绕过等。

- 🚨 **会话交叉污染**：由于错误使用 AsyncLocalStorage，并发请求可能导致用户会话数据泄露，使一个用户访问到另一个用户的认证信息。
- 🧪 **缓存键缺失**：共享缓存未包含授权头信息，导致首个用户的缓存响应被错误地提供给所有后续请求，造成数据泄露。
- 🔀 **中间件绕过**：URL 解码不一致使攻击者可通过双重编码路径绕过身份验证中间件，直接访问受保护页面。
- 🖼️ **图像优化器漏洞**：图像处理端点存在开放重定向问题，且失败时可能直接返回原始文件，绕过访问控制。
- 🛡️ **API 路由无保护**：默认配置下，API 路由被静默排除在中间件保护之外，导致关键接口未经验证即可访问。

### [理解 React Fiber 存在的意义 | React 内部探秘](https://inside-react.vercel.app/blog/understanding-why-react-fiber-exists)

**原文标题**: [Understanding Why React Fiber Exists | Inside React](https://inside-react.vercel.app/blog/understanding-why-react-fiber-exists)

React Fiber 是为了解决 React 15 中递归渲染导致的性能问题而引入的架构，它通过将渲染工作拆分为可中断和可恢复的小单元，实现了时间切片和优先级调度，从而提升用户体验。

- 🧵 **摆脱 JavaScript 调用栈限制**：React 15 使用递归渲染，一旦开始就无法中断，导致长时间任务阻塞用户交互，如输入延迟。
- ⏸️ **实现可中断的渲染过程**：Fiber 将渲染分解为小工作单元，每处理约 5 毫秒后主动让出控制权，允许浏览器处理高优先级事件（如点击、输入）。

- 🔄 **保持 React 核心哲学**：Fiber 在底层优化调度，但开发者仍使用纯函数组件，维持了状态驱动 UI 的简单心智模型。
