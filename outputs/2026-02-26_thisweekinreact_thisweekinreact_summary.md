### [Node.js 是单线程的吗？](https://nodesource.com/blog/is-nodejs-single-threaded-or-not)

**原文标题**: [Is Node.js Single-Threaded… or Not?](https://nodesource.com/blog/is-nodejs-single-threaded-or-not)

Node.js 的 JavaScript 执行默认是单线程的，但其运行时环境（包括 libuv 和操作系统）实际上使用了多线程来处理 I/O 和计算密集型任务，从而实现并发和高性能。

- 🧵 **JavaScript 执行默认单线程**：V8 引擎只有一个主调用栈，一次只能执行一段 JavaScript 代码，除非显式使用 Worker Threads 或进程。
- ⏱️ **异步操作由 libuv 管理**：例如 `setTimeout` 的计时器由 libuv 处理，回调仍在主线程执行，不阻塞其他代码。
- 🔄 **计算密集型任务使用线程池**：如 `crypto.pbkdf2` 会被委托给 libuv 的线程池，在后台并行计算，主线程保持空闲。
- 🧠 **区分执行与运行时**：Node.js 运行时是多线程的，但 JavaScript 代码本身默认不在主线程并行执行。
- 💡 **更实用的心智模型**：关注代码执行位置——是在主调用栈、libuv、操作系统还是显式并行中，这比简单标签更有助于理解。
