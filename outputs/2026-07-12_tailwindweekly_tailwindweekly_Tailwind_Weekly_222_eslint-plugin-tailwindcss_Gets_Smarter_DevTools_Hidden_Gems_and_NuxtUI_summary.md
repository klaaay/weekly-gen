### [让界面体验更佳的细节](https://jakub.kr/writing/details-that-make-interfaces-feel-better?ref=tailwindweekly.com)

**原文标题**: [Details That Make Interfaces Feel Better](https://jakub.kr/writing/details-that-make-interfaces-feel-better?ref=tailwindweekly.com)

本文總結了多個提升介面體驗的細節技巧，強調優秀設計源於細微之處的累積。

- 📝 **文字換行**：使用 `text-wrap: balance` 讓標題文字均勻分佈，`text-wrap: pretty` 防止段落末尾孤詞，兩者常搭配使用。
- 🔵 **同心圓角**：外層圓角半徑等於內層圓角半徑加上內邊距，確保嵌套元素視覺平衡。
- 🎨 **情境化圖示動畫**：對圖示應用 `opacity`、`scale` 和 `blur` 動畫，讓轉場更流暢且響應迅速。
- 🔍 **文字清晰化**：在 macOS 上使用 `-webkit-font-smoothing: antialiased` 讓文字渲染更細緻清晰。
- 🔢 **使用等寬數字**：對數字應用 `font-variant-numeric: tabular-nums`，避免數字更新時因寬度變化而跳動。
- ⏸️ **動畫可中斷**：優先使用 CSS 過渡而非關鍵幀動畫，讓使用者能隨時中斷互動，避免介面卡頓。
- 🧩 **拆分與交錯進場元素**：將進場動畫拆分為小塊（如標題、描述、按鈕），並加入延遲交錯，提升視覺層次感。
- 🚪 **退場動畫低調**：退場動畫應比進場更柔和，例如僅使用 `opacity` 和 `blur`，避免過度吸引注意。
- 👁️ **光學對齊而非幾何對齊**：當圖示與文字並排時，根據視覺感受調整邊距，而非嚴格按幾何邊界對齊。
- 🌑 **用陰影代替邊框**：使用多層 `box-shadow` 營造深度，比純色邊框更適應不同背景。
- 🖼️ **為圖片添加輪廓**：在圖片上添加 1px 半透明輪廓（黑或白），增加立體感與一致性。
- ⏱️ **前後對比**：應用上述技巧後，相同內容的卡片體驗明顯提升，可透過計時器、複製連結等互動驗證。
- 🛠️ **技能工具**：這些技巧已整合為開源技能，可透過 `npx skills add jakubkrehel/make-interfaces-feel-better` 安裝，並在專案中使用 `/skill` 指令調用。

### [什么是CSS包含性以及如何使用它？——CSS Wizardry](https://csswizardry.com/2026/04/what-is-css-containment-and-how-can-i-use-it/?ref=tailwindweekly.com)

**原文标题**: [What Is CSS Containment and How Can I Use It? – CSS Wizardry](https://csswizardry.com/2026/04/what-is-css-containment-and-how-can-i-use-it/?ref=tailwindweekly.com)

### 概述总结
CSS 包含是一种让浏览器将页面部分视为独立岛屿的技术，从而限制布局、绘制、尺寸或样式计算的范围，提升性能。

- 📐 **布局包含 (contain: layout)**：将内部布局与页面其他部分隔离，创建独立的格式化上下文和堆叠上下文，但需注意固定定位元素的行为变化。
- 🎨 **绘制包含 (contain: paint)**：将绘制限制在元素的内边距框内，溢出部分会被裁剪；若元素在屏幕外，浏览器可跳过其子元素的绘制。
- 📏 **尺寸包含 (contain: size)**：使元素尺寸脱离子元素影响，需配合显式尺寸或 `contain-intrinsic-size` 使用，否则可能导致零高度容器。
- 🔢 **样式包含 (contain: style)**：阻止 CSS 计数器等样式副作用扩散到子树外，但并非通用 CSS 作用域。
- ⚡ **特殊值：`content` 和 `strict`**：`content` 是 `layout paint style` 的简写，安全且适合多数场景；`strict` 包含所有四种类型，更强大但需谨慎使用。
- 🚀 **实际应用案例**：在 OpenTable 的移动抽屉菜单中，使用 `contain: strict` 将布局操作从影响 4,371 个节点（11.21ms）降至仅 73 个节点（1.89ms），性能提升约 6 倍。
- 🖼️ **`content-visibility` 与 `contain-intrinsic-size`**：`content-visibility: auto` 可跳过屏幕外内容的布局和绘制，需配合 `contain-intrinsic-size` 提供占位尺寸，适合长列表和重复内容。
- ⚠️ **注意事项**：可能引发零尺寸盒子、意外裁剪、新堆叠上下文等问题；需避免在根级包装器、小内联元素或依赖溢出的组件上使用。
- ✅ **使用建议**：优先对独立卡片、文章预览、第三方小部件使用 `contain: content`；对长列表使用 `content-visibility: auto`；谨慎使用 `contain: strict` 和 `contain: size`。
