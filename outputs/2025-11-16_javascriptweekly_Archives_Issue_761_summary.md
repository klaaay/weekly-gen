### [JavaScript 安全编码 - Stack Overflow](https://stackoverflow.blog/2025/10/15/secure-coding-in-javascript/)

**原文标题**: [Secure coding in JavaScript - Stack Overflow](https://stackoverflow.blog/2025/10/15/secure-coding-in-javascript/)

JavaScript 作为互联网前端核心技术，因其广泛使用而成为攻击者的主要目标。本文提出十项安全编码建议以增强 JavaScript 应用的安全性。

- 🛡️ 防范跨站脚本攻击（XSS）：通过输入验证、输出编码、内容安全策略（CSP）及安全 Cookie 设置进行防护
- ⚙️ 选用自动编码框架：优先使用 React、Angular、Vue.js 等具备自动输出编码功能的框架
- 📜 避免内联脚本：将 JavaScript 代码置于独立外部文件以降低 XSS 风险
- 🚀 启用严格模式：通过严格模式提升代码安全性、可预测性及性能
- 🔧 善用开源工具：采用 DomPurify、Retire.js、npm audit 等工具辅助安全检测
- ✍️ 明确文本标识：使用 innerText/textContent 替代 innerHTML 确保内容不被解析执行
- 🎯 限定安全属性：仅将用户数据设置于 align/class 等静态属性，避免动态事件属性
- 🔒 后端输入验证：关键安全校验必须在服务端执行，防止代理拦截篡改
- ⚠️ 规避危险函数：禁用 eval()、document.write() 等高风险函数处理用户数据
- 📋 全周期安全实践：将威胁建模、代码审查、安全测试纳入开发生命周期

通过持续践行这些安全准则，可逐步构建更健壮的 JavaScript 应用生态。
